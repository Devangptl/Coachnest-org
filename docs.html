A modern, full-stack online learning platform with course management, gamification, community, and Stripe payments.

CoachNest (v0.2.0) is a production-ready learning management system that supports:

  • ✓ Course creation & delivery — structured courses with sections, video/text/quiz lessons, progress tracking, and PDF certificates
  • ✓ Gamification — XP points, levels, badges, daily streaks, and a global leaderboard
  • ✓ Community — discussion forums, study groups, peer review submissions, and activity feed
  • ✓ Payments — Stripe Checkout with coupon support, webhooks, and refund management
  • ✓ Admin panel — full control over courses, students, orders, analytics, blogs, and messages
  • ✓ Email — transactional emails via Resend (welcome, purchase confirmation, course updates)
  • ✓ Multi-role auth — STUDENT / INSTRUCTOR / ADMIN with JWT + Edge-compatible sessions

🚀 Quick Start

Prerequisites

  • → Node.js 18+
  • → MySQL 8.0+ (required for full-text search)
  • → Stripe account (test mode)
  • → Resend account (for emails)

Installation

# Clone & install
git clone <repo-url>
cd learning
npm install

# Configure environment
cp .env.example .env
# → Edit .env with your values

# Set up database
npm run db:push
npm run db:seed
npm run db:seed:blogs

# Start dev server
npm run dev
ℹ️ The app starts at http://localhost:3000. The first user you register will have the STUDENT role. Promote them in the database or via prisma studio for admin access.

🔐 Authentication

Sessions use JWTs signed with JWT_SECRET and stored in an HTTP-only cookie named session. The jose library is used instead of jsonwebtoken to maintain compatibility with Next.js Edge Runtime.

Key Functions — lib/auth.ts

FunctionDescription

verifyToken(token)Verify and decode a JWT
setSessionCookie(res, token)Write JWT to HTTP-only cookie

🎮 Gamification System

Students earn XP for learning activities. XP accumulates into levels and unlocks badges. Daily activity maintains a streak.

⭐
XP Points
Awarded for completing lessons, passing quizzes, joining groups, writing reviews, and more. Logged to XpEvent.
📊
Levels
getLevelForXp(xp) maps total XP to a level. Level thresholds increase progressively.
🏅
Badges
Defined in BADGE_MAP. Awarded via grantBadge() on milestones. Stored in UserBadge.
🔥
Streaks
Daily learning streaks tracked via updateStreak(). Current and longest streak stored on UserGameProfile.
💡 The leaderboard at /api/gamification/leaderboard ranks users by total XP and is displayed on /dashboard/achievements.

💰 Payment Flow

1
Student clicks "Enroll"
Optionally enters a coupon code. POST /api/coupons/validate returns the discounted price.
2
Create Stripe Checkout Session
POST /api/payments/create-order creates a Stripe Checkout Session and returns its URL.
3
Student pays on Stripe
Stripe hosts the checkout page. The student enters payment details and confirms.
4
Verify & Enroll
Stripe redirects to /api/payments/verify?session_id=... which confirms payment, creates an Enrollment and Order record, and sends a purchase email.
5
Webhook (async)
POST /api/payments/webhook handles Stripe events asynchronously — payment success, refunds, disputes.
⚠️ In production, configure your Stripe webhook endpoint to point to https://yourdomain.com/api/payments/webhook and set STRIPE_WEBHOOK_SECRET accordingly.

🚢 Deployment

The project is pre-configured for Vercel (a .vercel/ config directory is present).

Build Command

npm run build
# Runs: prisma generate && next build

Production Checklist

  • Set all environment variables in Vercel project settings
  • Provision a MySQL 8.0+ database (PlanetScale, Railway, AWS RDS, etc.)
  • Set DATABASE_URL to the production connection string
  • Set JWT_SECRET to a long random string (min. 32 chars)
  • Set NEXT_PUBLIC_APP_URL to your production domain
  • Run prisma migrate deploy on the production database
  • Configure Stripe webhook to https://yourdomain.com/api/payments/webhook
  • Set STRIPE_WEBHOOK_SECRET from the Stripe webhook dashboard
  • Verify your email sender domain in Resend and set EMAIL_FROM
  • Set RESEND_API_KEY
  • Configure Cloudinary credentials for image uploads
