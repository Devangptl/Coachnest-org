Phased Rollout

Phase 1 — Foundation
AI Tutor Chatbot
📅 Weeks 1–5

Build the core intelligence layer first — every other feature benefits from it.

🤖
Embedded Chat Widget
Floating AI assistant on every course page. Answers subject-specific questions using Claude API.
📚
Context-Aware Responses
AI reads the active lesson content and answers in context — no generic replies.
📝
Doubt History Log
Students revisit past questions. Instructors see common doubts to improve content.
🎯
Quiz Generator
AI auto-generates 5–10 MCQs from any lesson material on demand.
Implementation Steps
Integrate Claude API / OpenAI with a system prompt scoped to subject content
Build React/Vue chat component — reusable across web & mobile
Store conversation history per student in backend (Node/Supabase)
Add lesson context injection (pass current lesson text to API)
Deploy widget as shared module — HTML embed for legacy pages, npm for React/Vue
Claude API React Component Supabase / Firebase REST API
Phase 2 — Engagement
Gamification System
📅 Weeks 6–10

Make learning addictive with structured reward mechanics tuned for academic behaviour.

⚡
XP & Level System
Earn XP for completing lessons, quizzes, and helping peers. Levels unlock new features.
🏅
Badges & Achievements
30+ milestone badges — First Question, 7-Day Streak, Top Scorer, Helpful Peer, etc.
🔥
Daily Streaks
Login and activity streaks with push notifications. Streak shields to protect progress.
🏆
Class Leaderboards
Weekly leaderboards scoped to class/grade — not global — to keep competition healthy.
Implementation Steps
Design XP schema: action → points table (lesson complete = 50 XP, quiz = 100 XP…)
Build gamification microservice (or Supabase Edge Function) to process events
Create badge definition JSON + badge unlock trigger system
Build animated XP bar + level-up modal in React/Vue (reuse on mobile)
Integrate push notifications (Firebase Cloud Messaging) for streak reminders on mobile
Add leaderboard API endpoint, scoped by class_id
Supabase Events Firebase FCM Framer Motion React Native Notifications
Phase 3 — Community
Social & Collaboration
📅 Weeks 11–16

Turn isolated learners into a connected academic community with peer-driven learning tools.

💬
Discussion Forums
Per-lesson threaded Q&A. Upvote best answers. AI flags unanswered questions for teachers.
📖
Study Groups
Students form groups by subject. Shared notes, group progress tracking, and group XP.
👨‍🏫
Peer Review System
Submit assignments for anonymous peer review. Structured rubric-based feedback forms.
📣
Activity Feed
Class feed showing badges earned, questions answered, and milestones — builds social proof.
Implementation Steps
Build forum data model: threads → replies → votes (PostgreSQL / Firestore)
Real-time replies using Supabase Realtime or Socket.io
Study group creation flow with invite link + shared workspace
Peer review matching algorithm (assign reviewers randomly, avoid self-review)
Activity feed with event aggregation — batch "3 students earned a badge" style updates
Mobile notifications for replies, group activity, and peer review assignments
Supabase Realtime Socket.io PostgreSQL Flutter Notifications
